# 抖医 TiktokRx - 产品需求文档

> 抖音短视频内容智能诊断平台 | 基于 Multi-Agent 辩论引擎

---

## 1. 产品概述

### 1.1 产品定位

抖医是一个基于 AI 多智能体辩论引擎的抖音内容诊断平台。用户提交抖音视频、图文作品、纯文字文章或背景音乐，5 位 AI 专家通过三轮辩论给出量化诊断报告、可执行优化方案、模拟评论区预测。

### 1.2 核心价值

| 传统方案 | 抖医 |
|----------|------|
| 凭经验主观判断 | 874 条真实视频 + BGM 数据驱动 |
| 单模型单次打分 | 5 Agent 并行诊断 → 交叉辩论 → 裁判综合 |
| 不分析 BGM | BGM 热度分析 + 适配度评估 + 推流建议 |
| "提升内容质量" | "BGM 应选择 S 级热门音乐，建议替换当前 BGM" |
| 无评论预测 | AI 模拟真实评论区（含质疑/杠精/求同款） |

### 1.3 目标用户

- **抖音创作者**：想提升内容质量但缺乏数据反馈
- **MCN 机构**：批量诊断旗下账号内容
- **品牌方**：评估达人合作内容质量
- **运营团队**：建立内容质量评估标准

## 2. 功能需求

### 2.1 核心功能

#### F1: 多模态输入
- 支持抖音分享链接解析（视频/图文/纯文字）
- 支持视频文件直接上传（MP4，最大 300MB）
- 支持图文作品截图上传
- 支持纯文字文章输入
- 支持 BGM 片段识别

#### F2: AI 视频解析
- 自动提取视频首帧作为封面
- OCR 识别视频内文字/字幕
- 关键帧提取分析画面
- 音频转写（支持普通话/方言）

#### F3: BGM 背景音乐分析（新增）
- 音频指纹识别当前 BGM
- 自动匹配抖音热门 BGM 曲库
- 输出 BGM 热度等级（S/A/B/C）
- 推流影响分析
- BGM 与内容适配度评估
- 热门 BGM 推荐

#### F4: 纯文字图文分析（新增）
- 文字情绪分析（正面/负面/中性）
- 关键词提取与标签推荐
- 排版结构分析（段落/emoji/符号）
- 互动引导词检测
- 文字节奏评估

#### F5: 实时辩论动画
- 11 步诊断时间线展示
- Agent 辩论实况气泡动画
- 实时状态跟踪（等待中/诊断中/辩论中）

#### F6: 七维雷达评分

| 维度 | 权重 | 说明 |
|------|------|------|
| 内容质量 | 25% | 脚本、图文排版、文字节奏 |
| 视觉表现 | 20% | 封面吸引力、画面质量、字幕 |
| **BGM 适配** | **15%** | 音乐热度、适配度、推流影响 |
| 增长策略 | 15% | 标签精准度、发布时机、互动引导 |
| 用户共鸣 | 10% | 情感连接、共鸣点、评论预测 |
| 技术表现 | 8% | 画质、音频质量、字幕准确性 |
| 综合评分 | 100% | 七维加权总分 |

#### F7: 完整诊断报告
- 七维雷达图可视化
- 各维度得分与详细评语
- **BGM 分析专项报告**
- 与同品类基线数据对比
- 优先级排序的优化建议
- 模拟评论区预测（含杠精/求同款/质疑）

#### F8: BGM 优化建议（新增）
- 当前 BGM 热度评估
- 替代 BGM 推荐（S/A/B 级）
- 适配品类推荐
- 换 BGM 后的预估推流变化

#### F9: 迭代优化引擎
- 自动生成 3 个高分改写方案
- 脚本优化建议（前置钩子/增强悬念）
- 封面方向建议
- 标签组合优化
- **BGM 替换方案**

#### F10: 分享卡片
- 一键生成带品牌水印的诊断卡片
- 支持导出为图片
- 支持系统分享

#### F11: 诊断历史
- 本地 IndexedDB 存储
- 支持按时间/品类/类型筛选
- 隐私安全（不上传服务器）

### 2.2 内容类型支持

|------|------|----------|-------------|
| 短视频 | 视频链接/上传 | 脚本+视觉+BGM+增长 | 七维雷达评分+BGM推流影响 |

## 4. AI 智能体设计

### 4.1 ContentAgent（内容分析师）

**职责**：评估视频脚本质量、图文排版、文字节奏

**诊断维度**：
- 开头钩子：是否在 3 秒内抓住注意力
- 脚本结构：起承转合是否清晰
- 图文比例：图片与文字的搭配是否合理
- 文字节奏：段落长短、emoji 使用、符号节奏
- 价值传递：是否提供实用信息/情感共鸣
- 结尾引导：是否有互动引导（关注/评论）

**输出格式**：
```
内容质量得分: 8.2/10
优点: 开头钩子强，第5秒即抛出悬念
缺点: 图文比例偏少，文字描述过于冗长
建议: 增加图片数量，删减第3段文字
```

### 4.2 VisualAgent（视觉诊断师）

**职责**：评估视觉呈现

**诊断维度**：
- 封面吸引力：首帧是否传递核心信息
- 画面质量：清晰度、稳定度、构图
- 字幕设计：字体大小、位置、可读性
- 色彩风格：与内容调性是否一致
- 信息密度：画面信息量是否适中

**输出格式**：
```
视觉得分: 7.5/10
优点: 封面主体突出，色彩明亮
缺点: 字幕过小，第8秒处被遮挡
建议: 放大字幕字号，添加背景色块
```

### 4.3 BGMAgent（BGM分析师）【新增】

**职责**：评估背景音乐与内容匹配度、热度

**诊断维度**：
- BGM 热度：当前音乐属于哪个热度等级（S+/S/A/B/C）
- 适配度：音乐风格与内容调性是否匹配
- 推流预测：基于 BGM 热度的推流影响评估
- 替代推荐：同风格但更高热度的 BGM 推荐
- 情绪同步：音乐节奏与画面节奏是否匹配

**输出格式**：
```
BGM适配得分: 6.5/10
当前BGM: "阳光男孩" - 热度指数 52,000 (A级)
优点: 音乐风格与美食内容调性匹配
缺点: 热度偏低，S级BGM推流效果会更好
推流影响: +8% (A级BGM加权)
建议替换:
  1. "快乐厨房" - 热度 128,000 (S级) ⭐推荐
  2. "美食时光" - 热度 89,000 (S级)
  3. "烹饪达人" - 热度 45,000 (A级)
预估换BGM后推流提升: +12%
```

### 4.4 GrowthAgent（增长策略师）

**职责**：标签策略与增长潜力

**诊断维度**：
- 标签数量：是否符合品类最优（4-8个）
- 标签精准度：与内容相关性
- 标签热度：是否包含潜在热门标签
- 发布时机：建议发布时间段
- 互动引导：是否埋设互动触发点

**输出格式**：
```
增长策略得分: 6.8/10
优点: 标签数量适中，覆盖核心关键词
缺点: 缺少 #美食教程 类热门标签
建议: 添加 #懒人食谱 #5分钟快手菜
```

### 4.5 UserSimAgent（用户模拟器）

**职责**：模拟真实用户观看心理

**模拟用户类型**：
- 目标用户：视频针对哪类人群
- 心理路径：观看时的注意力曲线
- 流失节点：可能在哪个时间点划走
- BGM 反馈：用户对 BGM 的反应（"这音乐好洗脑"）
- 互动意愿：看完后是否愿意点赞/评论

**输出格式**：
```
用户共鸣得分: 7.9/10
用户画像: 25-35岁职场女性，寻求生活技巧
观看体验: 第1-15秒保持兴趣，第23秒开始无聊
BGM反馈: "这音乐好欢快，但有点吵"
模拟评论: "求同款！BGM是什么？" "收藏了"
```

### 4.6 JudgeAgent（综合裁判）

**职责**：整合五维诊断，输出最终报告

**裁判逻辑**：
1. 收集五个 Agent 的独立诊断
2. 按权重计算加权总分
3. 对标基线数据判断所处水平
4. 识别核心问题与优先改进项
5. 生成可执行优化方案（包括 BGM 替换建议）

**最终报告格式**：

### 5.2 BGM 热度等级

| 等级 | 热度指数 | 颜色标识 | 推流加权 |
|------|----------|----------|----------|
| S+ | 100万+ | 🔥🔥🔥 | +30% |

| A | 10-50万 | 🔥 | +15% |

## 7. 技术需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 2s |
| 视频解析 | < 30s |
| BGM 识别 | < 10s |
| AI 诊断 | < 60s |
| 报告生成 | < 10s |

### 7.2 支持格式

| 类型 | 格式 | 限制 |
|------|------|------|
| 视频上传 | MP4 | 最大 300MB |
| 链接解析 | 抖音分享链接 | 短链/长链均可 |
| 图片 | PNG/JPG | 最大 10MB |
| 纯文字 | TXT | 最大 2000 字 |
| 音频片段 | MP3/WAV | 最大 5MB |

### 7.3 安全要求

- 视频/音频文件仅本地处理，不上传服务器
- BGM 识别调用第三方 API（仅传输音频指纹）
- 诊断历史存储在浏览器 IndexedDB
- API 密钥存储在本地 .env 文件
