# 抖医 TiktokRx - 技术架构

> 基于 NoteRx 多智能体辩论引擎的抖音内容诊断平台

## 产品定位

**抖医**是一个基于 AI 多智能体辩论引擎的抖音内容诊断平台。用户提交抖音视频链接、截图、图文或音频，5 位 AI 专家通过三轮辩论给出量化诊断报告、可执行优化方案、模拟评论区预测。

### 2. 视频解析模块

```
视频输入 → FFmpeg 首帧提取 → OpenCV 关键帧 → OCR 文字识别
                        ↓
              音频 → 音频指纹识别 → BGM 数据库匹配 → 热度分析
```

**输入支持：**
- 抖音视频链接（分享链接/短链）
- 视频文件上传（MP4 最大 300MB）
- 图文作品截图
- 纯文字文章（抖音图文）
- 背景音乐片段识别

**BGM 热度等级：**
| 等级 | 热度指数 | 推流影响 |
|------|----------|----------|
| S+ | 100万+ | 强力推流 |
| S | 50-100万 | 积极推流 |
| A | 10-50万 | 正常推流 |
| B | 1-10万 | 轻度推流 |
| C | <1万 | 不影响 |

### 4. 多智能体辩论引擎

**Round 1: 并行独立诊断（5 Agent）**
- ContentAgent（内容分析师）：评估脚本质量、图文排版、文字节奏
- VisualAgent（视觉诊断师）：封面吸引力、画面质量、字幕设计
- GrowthAgent（增长策略师）：标签策略、发布时机、互动引导
- BGMAgent（BGM分析师）：背景音乐适配度、热度、与内容匹配
- UserSimAgent（用户模拟器）：模拟真实用户观看心理

**Round 2: 交叉辩论**
- 内容 ↔ 视觉：图文比例是否合理
- 视觉 ↔ BGM：画面风格与音乐调性是否匹配
- 增长 ↔ BGM：BGM 热度是否能带动流量
- BGM ↔ 内容：音乐情绪与内容调性是否一致
- 用户 ↔ 内容：目标用户是否喜欢这个 BGM

**Round 3: 综合裁判**
- JudgeAgent 整合所有观点
- 给出最终评分（7 维度）
- 生成优化建议报告

## BGM 热度分析算法

```
BGM 热度得分 = 基础热度 × 适配度系数 × 时效性系数

其中：
- 基础热度 = 近7天播放量 + 收藏量×3 + 使用量×5
- 适配度系数 = 内容品类匹配度 × (0.8 + 情绪匹配度×0.2)
- 时效性系数 = 0.7 + (发布时间/7天)×0.3

推流影响：
- S级BGM: +30% 推流加权
- A级BGM: +15% 推流加权
- B级BGM: +5% 推流加权
- C级BGM: 无影响
```

## 快速开始

```bash
# 克隆
git clone https://github.com/jiangmuran/noterx.git && cd noterx

# 配置
cp .env.example backend/.env # 编辑填入 API Key

# 安装依赖
make install

# 初始化数据（包括 BGM 基线）
make data

# 启动服务
./start.sh
```

访问 `http://localhost:5173`
