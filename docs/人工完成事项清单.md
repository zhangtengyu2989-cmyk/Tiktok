## 1. API Key 配置

| 项目 | 详情 |
|------|------|
| **做什么** | 在 `backend/.env` 中填写真实的 API Key |
| **涉及字段** | `OPENAI_API_KEY`（必须）、`OPENAI_BASE_URL`、`LLM_MODEL_PRO` / `LLM_MODEL_FAST` / `LLM_MODEL_OMNI` |
| **模板** | 复制 `.env.example` 到 `backend/.env`，按注释填写即可 |
| **风险** | 没有 Key 则后端所有 AI 功能不可用，前端自动回退到 fallback 演示数据 |

## 4. 可选增强（如时间允许）

### 4.1 CLIP 视觉语义集成

- 选择 `open_clip` 本地模型或云端 embedding API
- 集成到 `backend/app/analysis/video_analyzer.py` 和 `image_analyzer.py`
- 需要评估模型体积对部署的影响

### 4.2 本地 OCR 备选（PaddleOCR）

- 在 `backend/app/analysis/ocr_processor.py` 中添加本地 OCR 分支
- 仅在 API 不可用或成本受限时启用
- 需要安装 PaddleOCR 依赖（体积较大）

### 4.3 Token 与成本优化

- 根据后端日志中的 token 统计（orchestrator 已自动记录），决定是否：
  - 将非裁判 Agent 降级为 `MODEL_FAST`（mimo-v2-flash）
  - 减少辩论轮为 2 Agent 对抗 + 裁判
  - BGM Agent 仅在视频类型时启用，图文/纯文字跳过
- 这些是架构决策，需要人工判断
